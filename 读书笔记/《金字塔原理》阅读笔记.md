## 第1篇 表达的逻辑

### 第2章 金字塔内部的结构

不要幻想坐下来就开始构想金字塔，首先你必须梳理你想要表达的思想。

加快你梳理思想的过程：

- 主题与子主题之间的**纵向关系**：疑问/回答式对话
- 各子主题之间的**横向关系**：归纳或演绎
- 序言的叙述方式：讲故事

#### 纵向关系

​	纵向关系上，上级是下级的抽象。实际操作上，是下级对上级主题中疑问的回答。金字塔结构的巨大价值就在于它迫使你在理清思路时，从视觉上使纵向的疑问/回答式对话关系清晰化。所有文章都应当具有疑问/回答式对话结构。

​	你可能会对这种对话关系中，作者的回答持不同意见，但是你能够基于此结构，清晰地了解整个表述的原因。之所以清晰，是因为其思想的组织一直紧紧围绕着回答由主题引发的读者的疑问。

#### 横向关系

​	横向关系上，表述必须具备明确的归纳或演绎关系，但不可同时具备归纳和演绎关系。归纳是相互关联的组合项；演绎关系为承前启后的论述，一般为3段式，分别为：通用结论，主/谓表述，推理结论。例如：所有人都会死，苏格拉底是个人，因此苏格拉底会死。

#### 序言结构

​	通过**讲故事**的序言来确定初始问题。通过纵向关系能够建立疑问/回答式的对话结构，但总要让读者有一个与主题相关的初始问题，来引起读者兴趣。

​	序言以背景、冲突、疑问、回答，引导读者了解你的思维过程。其中疑问是写作的动机。

- 背景：介绍背景。时间、地点、人物
- 冲突：指出冲突。在背景下发生了某件事
- 疑问：引出疑问。在此冲突下的疑问
- 回答：给出答案。针对疑问做答复

### 第3章 如何构建金字塔

可通过自上而下 或 自下而上构建文章金字塔结构，自上而下通常比自下而上更容易。

#### 自上而下

- 提出主题思想（客户PolarDB-O容灾）
- 设想受众的主要疑问（是否可行：可行）
- 写序言：背景-冲突-疑问-回答（背景：需要具备数据复制的服务；冲突：DTS支持PolarDB-O迁移；疑问：方案是否可行；回答：不可行）
- 与受众进行疑问/回答式对话（接上：为什么不可行；回答：xxx）
- 对受众新疑问，重复疑问/回答式对话

#### 自下而上

- 列出你想表达的所有观点
- 找出各观点之间的逻辑关系
- 得出结论

### 第4章 序言的具体写法

序言是通过讲故事的方式，向读者说明已知的信息：说明某种“背景”，这种“背景”中发生了某种“冲突”，从而引发了某种“疑问”，而整篇文章的目的就是回答该“疑问”。

- 背景：锁定特定时间和地点，传达出与**主题相关**，并且是**读者了解并认可**的信息，同时**引起读者兴趣**。比如：容灾必须要具备数据服务组件。
- 冲突：推动“故事”发展，能够促进读者提出“疑问”；4种冲突-疑问方式

| 背景（关于文章公认事实） | 冲突（推动故事发展，引发疑问） | 疑问                     |
| ------------------------ | ------------------------------ | ------------------------ |
| 需要完成某项任务         | 发生了阻碍完成任务的事情       | 应该怎么做？             |
| 存在某个问题             | 知道解决问题的方案             | 如何实施解决方案？       |
| 存在某个问题             | 有人提出了一个解决方案         | 方案是否可行？           |
| 采取了某项行动           | 行动未达到预期效果             | 为什么没有达到预期效果？ |

#### 序言与结构

序言严格包含背景、冲突、疑问、回答4个部分(疑问有时可省略不写)，但顺序可变化，分为

- 标准式：背景、冲突、答案。举例：客户需要建设容灾，必须具备数据复制的服务，而DTS是可以支持PolarDB-O数据复制的，但通过调研确认方案是不可行的。
- 开门见山式：答案、背景、冲突。举例：经调研确认无法借助DTS建设PolarDB-O建设容灾，
- 突出忧虑式：冲突、背景、答案。
- 突出信息式：疑问、背景、冲突、答案。

序言的目的是“提示”读者而不是“告诉”读者某些信息。【辰雪】未理解这句话的含义

序言长度取决于读者和主题需要。

全篇文章中心都依赖读者提出的第一个疑问，这种疑问全篇只能有一个，如果存在两个，则肯定是有关联的。

#### 关键句

关键句是每段的tilte，要回答文章主题思想的引起受众的新疑问，还要呈现文章的框架结构。如果文章过长就应该列出关键句。

> 关键句切忌不要使用“背景”、“介绍”作为一章节标题，因为这种标题可能和其他标题无法归纳成一个抽象层次；而且，无法展示完成意思，无法形成明确的归纳或演绎论证关系。关键句应当是完成的思想，完成的句子。

#### 序言的例子

> 从书中可找到序言的例子

其中一例：

背景：人们相信使用X工具将获得Y

冲突：确实在使用X，但是只有其他人获得Y

疑问：为什么其他人获得Y

回答：因为使用了A+B+X。（引出新问题：使用这些工具如何获得Y；进而引出关键句：a使用标杆对比，以判断其产品或服务的相对效率；b应用业务管理工具，以判断提供各种产品或服务的实际成本）

序言和关键句引言的区别：序言目的是提醒读者**全文章主题**。第一个关键句是提醒该段落主题和全文主题关系，其他关键句是向读者说明该段落主题与前面的主题关系。

#### 序言常见模式

1. 我们应该做什么（What）：发出指示式、比较选择式
2. 我们应该怎么做（How）：发出指示式、解释做法式
3. 我们是否应该这样做（Whether）：请求支持式
4. 为什么会发生这种情况（Why）

**发出指示式**：告诉或要求某人做某事。例如：我们打算做X，需要你做Y，你会问应该如何做Y？或 你做X，必须按照Y方式进行，你会问什么是Y方式？

**请求支持式**：你们遇到了X问题，解决它需要1万美元，我应该审批吗？

**解释做法式**：向读者解释解决问题的方法/步骤。例如：系统是X，系统不能正常工作，如何改进使其正常工作。建议列出原流程，通过新流程比较差异，就能告诉你关键句应该怎么写。

**比较选择式**：如果问题不能够彻底解决，那么可以尝试一些方案提出建议，这是比较选择式的应用场景。产生的关键句有“为什么此方案优于其他方案”，也就是要说明选择方案C的主要原因，以及放弃方案A和B的主要原因。如果任何一种方案都无法达到你的目标，或没有可选方案，那么这时的“疑问”要么是“哪一个”，要么是“我们应该做什么”。

### 第5章 演绎推理与归纳推理

演绎推理：演绎是一种线性的推理方式，最终是为了得出一个由逻辑词“因此”引出的结论，该结论会被上一层次思想所概括。

归纳推理：将一组具有共同特点的事实、思想、观点归类分组，并概括其共性。

#### 演绎推理

演绎推理具备三段式：大前提、小前提、推导结论。例如：鸟会飞，喜鹊是鸟，所以喜鹊会飞。

三段式也可以是：出现的问题，问题的根因，解决问题的方案。

连环式演绎推理：当有连续的、相关的演绎推理，会使推理过程显得冗长且呆板，可以省略一些简单推理步骤，或在关键句层次上尽量避免使用演绎法论述，而是用归纳法取代，目的是通过归纳法将“结论”归类，使演绎推理降低到各自归类的“结论”中，并减少在演绎推理中插入其他干扰信息简化演绎推理复杂性。

> 描述如下内容，如果采用 目前存在的问题（问题一、问题二），产生问题的原因（原因一、原因二），解决措施（措施一、措施二）这样推理，需要读者在头脑里对应一、二关系，当数量增多，读者理解起来难度增大。
>
> 如果按照解决措施进行归类，再对每个结论分别演绎推理，理解起来会好得多。如问题一、原因一、措施一；问题二、原因二、措施

| 当前存在的问题 | 产生   | 解决问题的措施 |
| -------------- | ------ | -------------- |
| 存问题一       | 原因   | 结论一         |
| 存问题二       | 原因二 | 措施结论二     |

#### 归纳推理

归纳推理必须具备：

- 正确定义该组思想。找到一个能够表示该组所有思想共同点的名词。
- 准确识别并剔除该组思想与其他思想不相称的思想。

#### 演绎推理与归纳推理的区别

- 演绎推理，第二点是对第一点主语或谓语的论述。

- 归纳推理，同组中的思想具有类似的主语或谓语。

上述特点可以用于检查演绎或归纳是否合理。

## 第2篇 思考的逻辑

本篇介绍思想的排序、总结思想的方法。

### 第6章 逻辑顺序

归纳为一组的所有思想，应该具有某种逻辑顺序。避免遗漏思想，和检查思想是否应在同组。

逻辑顺序有：按时间顺序（前因后果，操作步骤）、按结构顺序（整体和部分）、按主次顺序

时间顺序：按照采取的行动顺序，依次表述达到某一结果必须采取的行动。

结构顺序：按照示意图、地图、图画或照片想象某事务时的顺序，如组织结构图、关键成功要素示意图等。要求各部分相互独立；所有部分完全穷尽。

主次顺序：按照每组思想具有相同特性的程度高低排序，先强后弱，先主要后次要。

### 第7章 概括各组思想

归纳推理中的思想：确保思想属于同一组，应抽象、提炼、概括思想精华，形成“总结句”。“总结句”避免使用“缺乏思想”的句子，例如”这里有3个问题“此类无思想的句子。

“缺乏思想”的句子可能使同一组的思想不完全。多花些时间提炼出正确的概括性思想，概括思想组时，应该指出采取行动后的结果，或从共性中得出某种结论，一般分为：

- 行动性思想：采取什么样的行动和取得的结果。任何一组相互独立、完全穷尽的行动性思想，加上其产生的结果，就能形成一个特殊的封闭体系。
  - 如何理顺行动性思想：明确描述各行动；找出行动间明显的因果关系组合并划分层级，每各层级控制在5个以内；概括行动结果或目标。
  - **每个行动和结果的措辞都必须非常明确。如果某一项结果表述不明确，你无法判断是否已经列出了达到这一结果的所有步骤**。
- 描述性思想：描述信息具有共同点。
  - 如何总结描述性思想：找出思想的共性；找出思想间的密切联系；完成归纳概括出主题。
  - 进行分组时要问自己“为什么只列出这些思想，这些思想的共性是什么，有无联系”，概括思想应指出采取上述思想的直接结果。

## 第3篇 解决问题的逻辑

本篇介绍如何处理“解决方案”这类的文章。这类文章一般有

- 我们应该做什么。（不知道解决方案）
- 我们应该做吗？（得到解决方案）
- 我们怎么做？（得到且接收解决方案）

这类场景的处理步骤：界定问题 -> 结构化分析 -> 搜集信息、分析问题 -> 解决方案 -> 与他人交流

其中界定问题和结构化分析这两部分时本篇重点介绍的，如果这两部分处理好，很容易写好文章。

### 第8章 界定问题

问题是现状和目标之间的差距。界定问题的方法称为连续分析，按逻辑顺序寻找答案：

| 问题                                                       | 归类         |
| ---------------------------------------------------------- | ------------ |
| 1、是否有/是否可能有问题（或机会）？ <br />2、问题在哪里？ | 界定问题     |
| 3、为什么存在（产生问题的根源、原因）                      | 寻找问题根源 |
| 4、我们能做什么？ <br />5、我们应该做什么？                | 寻找解决方案 |

如何描述问题：

讲述序幕 -> 对序幕产生影响的困扰（内部、外部） -> 受困扰引起的非期望现状R1 -> 描述最终期望目标R2

PS：解决问题的第一步就是确定R2

常见的问题：

1. 不知道如何从R1到R2。Q=我们应该做什么？
2. 知道R1到R2的解决方案，但不敢肯定是否正确。Q=我们是否应该做我们想做的事？
3. 知道R1到R2的正确解决方案，但不知道如何实施。Q=我们应该如何做我们想做的事？
4. 知道R1到R2的解决方案，且已经实施，但由于某种原因行不通。Q=解决方案行不通，我们应该做什么？（类型1的延伸）
5. 确定了好几个方案，不知道选哪一个。Q=我们应该选择哪种方案？
6. 知道R1，但不能具体描述R2，所以无法找到解决方案。Q=我们应该采取什么措施
7. 知道R2，但不清楚自己是否处于R1。Q=我们存在问题吗

书中对上述常见问题进行了抽象的举例，内容较多，可直接参考原书内容。

### 第9章 结构化分析问题

分析问题的流程是：收集信息、描述发现、得出结论、提出方案。

**收集信息**

强调收集信息要有的放矢，只收集和问题相关的资料。一种科学建议方法是只有假设自己已经找出产生问题的原因，才能更好的提出创造性解决问题的方案：

- 针对问题提出各种假设                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         
- 设计几项重要的实验，根据结果排查假设
- 通过实验得出明确结论
- 相应地采取补救措施

**设计诊断框架**

使用诊断框架可以实现分析问题可能的原因，帮助你设想产生问题的领域状况，从而提出假设。

例如按照第6章中，通过结构分析、因果分析、归类分析，找出问题原因。

结构分析：画一幅系统的现状或理想状况图，找到和分析产生问题的潜在原因。

因果分析：按照结构分析的结果，进一步按照因果逻辑分析根因。

**逻辑树**

建立逻辑树，对分析出的发现的问题，进行“是否问题分析”，得出结论。

### 第11章 在PPT演示文稿中呈现

设计PPT应掌握的基本原则：

- 幻灯片只包含最重要的、经过分组和总结的思想，叙述时应简洁
- 应图文并茂，使用各种图、表、示意图，更形象更直观，更具冲击力
- 应呈现经过深思熟虑的故事梗概和剧本（如序言、关键句要点、下一层次要点）
- 图表幻灯片占90%，文字幻灯片占10%。【辰雪】：个人认为这个不一定
- 文字幻灯片说明文稿的框架结构
- 文字幻灯片强调重要思想、观点、结论、论点、建议
- 图表幻灯片阐述文字难以说清楚的数据和关系
